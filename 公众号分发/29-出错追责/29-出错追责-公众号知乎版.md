# 同一个错犯了三次，我才决定自己写个系统管起来

> 风管厂出的错，十次有八次不是技术问题，是管理问题。

---

## 一、一个弯头的返工费，够我吃三个月盒饭

去年夏天，车间退回来一批弯头，包角全做错了——该用防火板的用了镀锌板，该做内菱角的没做。

我去追责，三个人互相推：

"我按图做的。"
"图是老刘给的。"
"老刘说不会画，让我问小王。"
"小王说这个单子的CAD是外包的。"

查了一圈，是报价阶段给的信息有遗漏，到生产环节已经转了三手。

最气人的不是花了6000多块返工——是三个月前同一个项目、同一个原因、同样的推诿链条，已经出过一次了。

## 二、为什么出错会重复发生

风管厂的出错，大多数不是技术问题，是这四点：

**信息传递断层**——报价→排版→下单→车间，每转一手都漏信息。

**责任界定模糊**——设计画错了图，排版没看出来算谁的？车间按错图做了呢？每个人都说"不是我一个人的问题"。

**没有纠错机制**——出了错，口头骂两句就过去了。下次同样的问题又冒出来。

**没有统计思维**——总觉得是偶然失误，没意识到某些工序的出错率本来就是系统性的。

我们试过用Excel记出错。每人一个文件夹，月底汇总。结果嘛——记不记看自觉，有人记了30条，有人3条——不是后者做得好，是前者老实。想分析哪类错最多，搜关键词搜得头皮发麻。

## 三、一个数据库系统怎么解决这事

去年下半年，技术部自己开发了一套管理系统（Delphi + MySQL），其中一个模块就是出错登记。分三层设计：

**第一层：记什么**

| 字段 | 说明 |
|------|------|
| 项目名称 | 哪个项目出的错 |
| 出错时间 | 精确到日 |
| 出错类型 | 尺寸/工艺/材料/图纸/沟通… |
| 责任成员 | 主要责任人 |
| 连带责任人 | 还有谁参与了 |
| 损耗成本 | 材质+平方+单价，系统自动算 |

每一条记录就是一个结构化的数据点，不再是"文本描述"。

**第二层：怎么联**

出错记录关联到三块：个人绩效（月底自动算）、项目档案（同一个项目的历史记录一键调）、类型统计（看到底哪类错最多）。

**第三层：怎么关**

查到原因不是结束，是开始：工艺标准不清晰→修订规范；沟通遗漏→改流程；系统Bug→修掉。

每条记录备注里都有一行"改进措施已实施"，什么时候改的、改了啥，一清二楚。

## 四、跑了快一年的效果

出错率从月均12~15次降到5~7次，重复出错从30%降到不到10%。

最意外的收获是：师傅们反而更愿意记录了。以前口头追责像审犯人，现在系统记录不带情绪。出错→登记→分析→改进，成了一个正常流程。

Q1统计结果：出错最多的是"尺寸标注不清"（占37%），我们就专门出了一个《图纸标注规范》培训课。

## 五、一张表，你也能起步

不用写代码，一个Excel就行：

| 需要记录的 | 为什么重要 |
|-----------|-----------|
| 项目名称 | 追查是否反复出错 |
| 出错类型 | 按大类归类 |
| 责任人+连带人 | 客观记录，不推诿 |
| 损耗金额 | 用数据说话 |
| 根因分析 | 是标准、流程还是人的问题？ |
| 改进措施+关闭日期 | 没闭环等于没记 |

每周花15分钟过一遍，三个月后你可能会发现——厂里80%的出错，只来自20%的原因。
