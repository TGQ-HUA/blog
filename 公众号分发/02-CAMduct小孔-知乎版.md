# 【知乎版】CAMduct 小孔不出 NC 码的解决方法（所有版本通用）

## 问题现象

使用 CAMduct 输出 NC 码时，发现**直径 ≤5mm 的小孔不生成切割代码**。大孔正常输出，只有小孔"消失"了。

这个问题不限于某个特定版本——从老版本到最新版都可能遇到，因为根因是同一个默认参数。

## 原因分析

经过排查，问题出在 CAMduct 的**穿孔尺寸（Pierce Size）**参数上。

这个参数的含义是：**小于等于该值的孔，不生成穿孔切割代码。**

系统默认值通常为 5.00mm，所以 5mm 及以下的孔就被"吞"掉了。

这个参数存在于所有版本的 CAMduct 中（包括 Autodesk Fabrication CAMduct），位置和逻辑完全一样。

## 解决方法

1. 打开 CAMduct
2. 进入 **File → Setup → Installed Machines**
3. 选择你的机床
4. 进入 **Tool Setup → Global** 面板
5. 找到 **Pierce Size（穿孔尺寸）** 参数
6. 将其改为 **0** 或 **1mm**
7. 保存，重新输出 NC 码

## 相关检查项

如果改了 Pierce Size 还是不行，继续检查：

- **Pierce Delay（穿孔延时）**：设为 0.1 秒。太长可能导致控制器跳过小孔
- **自动嵌套中的孔缝合值**：检查是否把相邻小孔"合并"了
- **切割工具配置**：确认机床已添加对应孔径的切割刀具

## 经验总结

CAMduct 的很多"诡异"行为，根源都在 Machine Setup 里。遇到输出异常，别急着怀疑图纸——**先去翻机床设置**，十有八九能找到答案。
