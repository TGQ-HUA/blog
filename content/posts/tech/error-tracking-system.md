## 引子：一个真实问题

去年夏天，车间退回来一批弯头，包角全做错了——该用防火板的用了镀锌板，该做内菱角的没做。

我去追责，三个人互相推：

"我按图做的。"
"图是老刘给的。"
"老刘说不会画，让我问小王。"
"小王说这个单子的CAD是外包的。"

查了一圈，最后发现是报价阶段给的信息有遗漏，到生产环节已经转了三手，没人说清楚。

最气人的不是这个错花了 6000 多块返工——是**三个月前同一个项目、同一个原因、一样的推诿链条，已经出过一次了**。

## 正文：技术分析

### 一、为什么出错会重复发生

风管厂的出错，十次有八次不是技术问题，是管理问题：

1. **信息传递断层**——报价→排版→下单→车间，每转一手都可能漏信息
2. **责任界定模糊**——设计画错了图是设计的事，排版没看出来呢？排版也有责？那车间的师傅按错图做了，有没有责任？
3. **没有纠错机制**——出了错，口头骂两句就过去了。下次还是同样的问题
4. **没有统计思维**——总觉得是"偶然失误"，没意识到某些工序的出错率是系统性的

我们之前也试过用 Excel 记出错——每人一个文件夹，月底汇总。结果：
- 记不记全看自觉
- 有些人一个月记了 30 条，有些人 3 条——不是后者做得好，是前者老实
- 想分析什么类型的错最多，Excel 里搜关键词搜得满头汗

### 二、一套数据库系统怎么解决

去年下半年，技术部自己开发了一套管理系统（Delphi + MySQL），其中一个模块就是**出错登记**。

这个模块分三层设计：

**第一层：记录什么（字段设计）**

| 字段 | 说明 |
|------|------|
| 项目名称 | 哪个项目出的错 |
| 出错时间 | 精确到日 |
| 出错类型 | 分大类（尺寸/工艺/材料/图纸/沟通…）|
| 责任成员 | 谁承担主要责任 |
| 连带责任人 | 还有谁也参与了 |
| 简要说明 | 一句话说清楚 |
| 详细报告 | 过程复盘 |
| 损耗成本 | 材质+平方+单价，系统自动算 |

这样每一条记录都是一个结构化的数据点，不再是"文本描述"。

**第二层：怎么关联（数据打通）**

出错记录关联到：
- **个人绩效**——月底算绩效的时候，出错次数和损耗金额自动算进去
- **项目档案**——同一个项目的历史出错记录可以一键调出来
- **类型统计**——按出错类型聚合，看到底哪类错最多

**第三层：怎么避免（闭环）**

查到原因不是结束，是开始：
- 如果是工艺标准不清晰 → 修订管件排版规范
- 如果是沟通遗漏 → 改流程：特殊工艺要求必须书面标注，不接受口头交代
- 如果是系统Bug → 修复

每次出错记录的备注里，都有一条"改进措施已实施"，什么时候改的、改了什么，一清二楚。

### 三、跑了半年的效果

这个模块上线到现在快一年了（2026年6月前），几个直观的变化：

1. **出错率**——技术部月均出错次数从上线前的 12~15 次降到现在的 5~7 次
2. **重复出错**——上线前有 30% 的错是同类问题，现在降到了不到 10%
3. **追责不费嗓子了**——打开数据库搜一下项目名，谁经手的、什么类型的错、什么原因，全在
4. **数据说话**——Q1 统计结果：出错最多的是"尺寸标注不清"（占 37%），我们就专门出了一个《图纸标注规范》培训课

最意外的收获是：**师傅们反而更愿意记录了**。

以前口头追责像"审犯人"，现在系统记录不带情绪。出错→登记→分析→改进，成了一个正常流程，没人觉得是故意找茬。

## 延伸：行业思考

风管制造是一个高度依赖"人"的行业。

师傅的技术靠积累，图纸的理解靠经验，信息的传递靠喊。但正因为这样，出错才不可避免。

我见过太多小厂的做法是：出错了骂一顿、扣点钱，完事。**治标不治本**。

我观察下来，比较管用的做法是：
- 承认出错是必然的（不是师傅们不认真）
- 用系统把出错变成数据
- 用数据分析找到规律
- 用规律指导流程改进

奇美星做这套系统的时候，花的最多的不是写代码的时间，是**想清楚要记什么、怎么归类、怎么反馈**。

## 实操：一张表搞定

如果你也想建一个简单的出错追查体系，不用写代码，一个 Excel 就能起步：

| 需要记录的 | 为什么重要 |
|-----------|-----------|
| 项目名称 | 追查相同项目是否反复出错 |
| 出错类型 | 按大类归类：尺寸/工艺/材料/图纸/沟通/其他 |
| 责任+连带责任人 | 客观记录，不推诿 |
| 损耗金额 | 用数据说话，别"感觉亏了" |
| 根因分析 | 是标准问题/流程问题/人的问题？ |
| 改进措施+关闭日期 | 没闭环的记录等于没记 |

每周花 15 分钟过一遍，三个月后你会有意外发现——**厂里 80% 的出错，可能只来自 20% 的原因**。

## 结尾：接地气的收束

现在排版室的墙上贴了一张纸，写着："出错不可怕，可怕的是同一个错犯第三次。"

上回有个新来的排版员做错了大小头的就料方向，自己查了系统记录，发现三个月前同样的事出了 3 次。他主动来找我："老唐，这个型号的大小头是不是该加个流程图？"

我说行，你写呗。后来那张流程图真贴出来了。
