## 四、推送上线命令

每次修改完内容后，执行以下命令：

```bash
cd /d G:\blog
git add .
git commit -m "更新说明"
git push
```

推送后 Cloudflare 自动重新编译，约 **1 分钟**后公网生效。

### 本地预览（推送前检查）

```bash
cd /d G:\blog
C:\tools\hugo\hugo.exe server --buildDrafts
```

浏览器打开 http://localhost:1313/ 查看效果，满意后再推送。

### 注意事项

- 推送前需开启 **Watt Toolkit** 的 GitHub 加速
- git 需关闭 SSL 验证：`git config --global http.sslVerify false`
- 如果 Token 过期：去 GitHub Settings → Tokens 重新生成

## 八、常见问题

| 问题 | 解决方法 |
|------|---------|
| git push 失败 | 开 Watt Toolkit GitHub 加速；检查 Token 是否过期 |
| 网站打不开 | `ipconfig /flushdns` 清缓存；换无痕模式；手机 4G 试 |
| 本地预览报错 | 检查 hugo.yaml 语法（YAML 缩进敏感） |
| 图片不显示 | 图片放 `static/img/`；引用路径 `/img/` 开头 |
| 搜索不工作 | 确认 search/index.md 有 `outputs: [HTML, JSON]` |
| 新文章不出现 | 检查 date 不是未来日期；draft 不是 true |
| SSL 证书错误 | 等 5~15 分钟让 Cloudflare 签发；清浏览器缓存 |
