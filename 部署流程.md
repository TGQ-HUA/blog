# 博客部署流程：Cloudflare Pages

> 目标：把 G:\blog 部署到公网，任何人通过浏览器即可访问
> 方案：Cloudflare Pages（免费 + CDN + 自动部署）
> 预计耗时：30~60 分钟

---

## 前置准备

你需要准备：
- 一个**邮箱**（注册 GitHub 和 Cloudflare 用）
- 能打开浏览器

---

## 第一步：注册 GitHub 账号（代码仓库）

1. 浏览器打开 https://github.com/signup
2. 填写邮箱、密码、用户名（建议用英文，如 `tangguoqi` 或 `tgq-liantai`）
3. 验证邮箱
4. 登录后**新建一个仓库**：
   - 点右上角 `+` → `New repository`
   - Repository name 填：`blog`
   - 选 `Public`（公开，Cloudflare Pages 免费版需要）
   - **不要**勾选 "Add a README"
   - 点 `Create repository`

> ⚠️ 如果 GitHub 打不开，告诉我，我帮你用镜像方案解决。

---

## 第二步：把博客代码推到 GitHub

在你电脑终端执行（我来帮你操作）：

```bash
cd /d G:\blog

# 初始化 git（如果还没有的话）
git init

# 添加所有文件
git add .

# 提交
git commit -m "初始化博客"

# 关联远程仓库（替换成你的用户名）
git remote add origin https://github.com/你的用户名/blog.git

# 推送
git push -u origin main
```

> ⚠️ 推送时需要登录 GitHub（会弹窗或要求输入 Token）。
> 如果 GitHub 推送不了（网络问题），我帮你配镜像加速。

---

## 第三步：注册 Cloudflare 账号

1. 浏览器打开 https://dash.cloudflare.com/sign-up
2. 填写邮箱 + 密码，注册
3. 验证邮箱
4. 登录进入控制台

---

## 第四步：创建 Cloudflare Pages 项目

1. 登录 Cloudflare 控制台后，左侧菜单点 **「Workers 和 Pages」**
2. 点 **「创建」** 按钮
3. 选择 **「Pages」** 标签页
4. 点 **「连接到 Git」**
5. 选择 **GitHub**，授权 Cloudflare 访问你的 GitHub 仓库
6. 选择刚才创建的 `blog` 仓库
7. 配置构建设置：

| 设置项 | 填写内容 |
|--------|---------|
| 项目名称 | `blog`（这会决定你的域名：blog-xxx.pages.dev） |
| 生产分支 | `main` |
| 框架预设 | 选 **Hugo** |
| 构建命令 | `hugo --gc --minify` |
| 构建输出目录 | `public` |

8. 点 **「保存并部署」**

---

## 第五步：等待部署完成

- Cloudflare 会自动从 GitHub 拉代码 → 安装 Hugo → 编译 → 部署
- 大约 1~2 分钟完成
- 完成后你会看到一个地址，类似：

```
✅ https://blog-abc.pages.dev
```

**这就是你的博客公网地址了！** 任何人都能访问。

---

## 第六步：验证

- 浏览器打开那个 `.pages.dev` 地址
- 确认首页、成果、知识库、文章都能正常显示
- 手机微信里打开试试，确认移动端也正常

---

## 以后更新博客的流程

```
写完文章 / 改完内容
  ↓
终端执行：
  cd /d G:\blog
  git add .
  git commit -m "新文章：xxx"
  git push
  ↓
Cloudflare 自动检测到更新 → 重新编译 → 30秒内上线
```

**就三条命令的事，跟本地预览一样简单。**

---

## 可选：绑定自定义域名

如果你后面想要自己的域名（如 `tangguoqi.com`）：

1. 在域名注册商（如阿里云/腾讯云/Namesilo）买一个域名（~50元/年）
2. 在 Cloudflare Pages 项目设置里点「自定义域」
3. 添加你的域名
4. 按提示去域名注册商那里改 DNS 指向 Cloudflare
5. 等 5~10 分钟生效

> 不买域名也完全没问题，`.pages.dev` 地址永久免费使用。

---

## 可能遇到的问题

| 问题 | 解决 |
|------|------|
| GitHub 打不开/推不了 | 用镜像：`git remote set-url origin https://ghfast.top/https://github.com/用户名/blog.git` |
| Cloudflare 构建失败 | 检查构建日志，大概率是 Hugo 版本问题，需要设置环境变量 `HUGO_VERSION=0.163.1` |
| 部署后页面空白 | 检查 `baseurl` 配置是否改成了实际的 pages.dev 地址 |
| 图片不显示 | 检查图片路径是否以 `/img/` 开头 |

## 操作顺序总结

```
① 注册 GitHub → 建仓库
② 推代码到 GitHub
③ 注册 Cloudflare → 连接仓库 → 配置构建
④ 等 2 分钟 → 拿到公网地址
⑤ 验证 → 完成！
```
